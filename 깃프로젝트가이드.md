# 깃허브 프로젝트 관리

1. Issues : 작업 단계별 업무와 서브업무 관리
2. Labels : 태그 느낌, 이슈 구분
2. Milestones : 개발 목표일(마감일) 관리
4. Projects : 전반적인 진행 현황 관리

## Issues
![issue](img/issue1.png)
- 작업 단계별 업무정의
- 체크리스트 작성
![issue2](img/issue2.png)
- 커밋할때 끝부분에 Issue 태그를 붙이면, 자동으로 커밋이 해당 Issue로 분류
```
git commit -m "docs: 이슈 사용법 작성 #1"
```
